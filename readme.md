üìä Step 3: Analysis and Observations

1. Passwords with greater length and character diversity achieve higher scores.

2. Randomness and unpredictability significantly increase strength.

3. Short or predictable passwords (e.g., names, words) are highly vulnerable.

4. Each added symbol or character type increases entropy (difficulty to crack).

üí° Best Practices for Creating Strong Passwords

a. Use at least 12‚Äì16 characters.

b. Combine uppercase, lowercase, numbers, and symbols.

c. Avoid personal information (names, dates, etc.).

d. Create unique passwords for every account.

e. Use passphrases (e.g., Blue$Tiger!Jump2025).

f. Store securely in a password manager.

g. Enable multi-factor authentication (MFA) for extra protection.

üß† Common Password Attacks
| **Attack Type**         | **Description**                    | **Mitigation**                 |
| ----------------------- | ---------------------------------- | ------------------------------ |
| **Brute Force**         | Tries all possible combinations    | Use long and complex passwords |
| **Dictionary Attack**   | Uses common words and patterns     | Avoid dictionary words         |
| **Phishing**            | Tricks users to reveal credentials | Verify URLs, enable MFA        |
| **Credential Stuffing** | Reuses leaked passwords            | Use unique passwords per site  |

Password Complexity and Security:
Password complexity directly impacts security. Longer and more diverse passwords exponentially increase the time and computing power needed to break them.
For example:

a. Simple password (8 chars): cracked in seconds

b. Complex password (16 chars with symbols): may take centuries to brute force
